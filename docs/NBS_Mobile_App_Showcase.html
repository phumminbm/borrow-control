01 / 05 · BR RETURN

Pick the items.
Just the ones.

Open any BR's detail, tap "Request Return", and the items are right there. Select what you want to return — Select All for the whole BR, or tap individually. Already-allocated quantities are preserved if you backtrack.

  • One-tap selectSingle line-items, or "Select all" for the entire BR.
  • Selected counterReal-time count in the bottom CTA — "Selected 2 →".
02 / 05 · BR RETURN

Multi-type allocation.
Per item.

Split a single line across RETURN, CLAIM, SALE, and FREE — exactly mirroring the Desktop BR Return flow. Steppers clamp at the source quantity so you can't over-allocate. The bulk-action bar fills every selected item with one tap.

  • Four types per itemRETURN (warehouse-bound), CLAIM (defect), SALE (post-sale), FREE (giveaway).
  • Bulk-fill"Select all as RETURN" — the quick path when one type dominates the BR.
  • Smart clampingCan't over-allocate. The stepper stops at the source row's available quantity.
03 / 05 · BR RETURN

Review.
Then submit.

Step 4 lays out the entire request — customer, BR, every item with its type breakdown chips, the running total, and your remark. One tap to submit, one tap to back out and fix.

  • Full reviewSubmit-total pill matches what Admin sees on the desktop — zero surprises.
  • Real-backend saveSubmit talks to /return-requests with a spinner. Success only when the row lands.
04 / 05 · BR RETURN

Submitted.
And tracked.

A full-screen success view confirms the new RT-T-* request ID. From here, jump back to BR list to file the next one for the same customer — or watch its status under the Returns tab.

  • Unique request IDTimestamp-based — collision-free even with concurrent submissions.
  • Live status30-second polling keeps the Returns tab in sync with Admin's desk.
05 / 05 · BR RETURN

Correction round-trip.
With evidence.

When Admin sends a request back for revision, you get the full context — which items, why, and photographic evidence. Edit just the flagged items and resubmit. Every round is preserved in the revision history.

  • Edit and resubmitOne-tap returns you to the qty step, scoped to just the flagged items.
  • Photo evidenceAdmin's attached photos open in a fullscreen lightbox with prev/next nav.
  • Rev N chip"Rev 2" badge surfaces every time, on every view, on both Sale and Admin sides.
The complete flow

Mobile to desk.
And back again.

Every return-request takes the same path. Sale on Mobile creates the request; Admin reviews on Desktop. Approvals sync back to Logistics File overnight. Rejections come back to the Sale's pocket — with reasons.

Built-in capabilities

Small details.
Big difference.

TH / EN

Full bilingual support. Toggle between Thai and Natural English in any view. Customer data and product names stay in their original language.

Dark / Light

iPhone-style sliding theme switch with a sunrise gradient. Persists per browser — flips with one tap from the topbar.

Real-time sync

30-second background polling keeps Mobile in sync with Desktop. Status changes surface within seconds of Admin's action.

Photo evidence

Real camera/library picker for proof. Auto-compressed to 600px and stored as base64 — the same shape Desktop uses.

Revision history

Every correction round captured with date, Admin remark, flagged items, reasons, and photos. Full audit trail per request.

Test mode

Submit real requests to the real backend with isTest=true. Visible only in Desktop's TEST queue. Apps Script writeback blocked. Clean cleanup query.

Optimistic UI

Submissions appear instantly in the Returns list. Rolls back cleanly if the backend rejects — you stay on Step 4 with a clear error and your form intact.

Calendar picker

Hotel-app-style date range with scroll-to-current month, Buddhist / Gregorian year switching, and a soft pink selection band.

Offline-ready cache

Customer and BR data cached on first load. Open the app on poor signal and still see your portfolio.

Quick start

How to use it.
End to end.

Five steps from "I just opened the app" to "Admin approved my return". Each one is one tap.

01

Pick your Sale name

First launch shows the Sale picker. Tap your name once — it's remembered for next time.

  • Organized by team for fast scanning
  • One tap to set, persists in browser storage
  • Switch later from the Profile tab
02

Browse customers · find the BR

Open the Customers tab. Search, filter by status, or sort by days outstanding. Tap a customer to see their full BR list.

  • Search by name or 5-digit customer code
  • Sort: Days · Value · Name
  • Tap a BR to drill into its items
  • Export the PDF file in the Borrow format
03

Request the return

From BR Detail, tap "Request Return". Pick items, allocate quantities across return types, add a remark, review, and submit.

  • 4-step flow with a back button on every screen
  • Smart clamping prevents over-allocation
  • "Select all as RETURN" for fast common cases
04

Wait for Admin · or get it back

Status appears in the Returns tab within 30 seconds of Admin's decision. Approved? Done. Sent back? Open the request to see why.

  • Background polling — no manual refresh needed
  • Pull-to-refresh anytime to force a check
  • Toast notifications surface backend errors
05

Revise and resubmit (if needed)

Rejected requests show Admin's note, the flagged items, the reasons, and any photo evidence. Tap "Edit and resubmit" to revise just the flagged items — approved items pass through automatically.

  • Scoped editor — only revise what Admin asked for
  • Rev 2 chip appears on every view, both sides
  • Full history viewer lets Admin audit every round
06

Customize the experience

Open Profile to flip between TH/EN, dark/light, change Sale, or check version info. Settings persist across sessions.

  • Language toggle: Thai · Natural English
  • Theme toggle: iPhone-style sliding switch
